Oracle database tables
Your CPRG 250 skills applied to a real product
-- Users table
CREATE TABLE users (
  user_id    NUMBER PRIMARY KEY,
  name      VARCHAR2(100) NOT NULL,
  email     VARCHAR2(100) UNIQUE NOT NULL,
  user_type VARCHAR2(10), -- 'poster' or 'worker'
  rating    NUMBER(3,2) DEFAULT 0,
  created_at DATE DEFAULT SYSDATE
);

-- Jobs table
CREATE TABLE jobs (
  job_id      NUMBER PRIMARY KEY,
  title       VARCHAR2(200) NOT NULL,
  description CLOB,
  budget      NUMBER(10,2),
  category    VARCHAR2(50),
  location    VARCHAR2(200),
  status      VARCHAR2(20) DEFAULT 'open',
  posted_by   NUMBER REFERENCES users(user_id),
  created_at  DATE DEFAULT SYSDATE
);

-- Applications table
CREATE TABLE applications (
  app_id     NUMBER PRIMARY KEY,
  job_id     NUMBER REFERENCES jobs(job_id),
  worker_id  NUMBER REFERENCES users(user_id),
  message    CLOB,
  price      NUMBER(10,2),
  status     VARCHAR2(20) DEFAULT 'pending',
  created_at DATE DEFAULT SYSDATE
);

-- Payments table
CREATE TABLE payments (
  payment_id   NUMBER PRIMARY KEY,
  job_id       NUMBER REFERENCES jobs(job_id),
  amount       NUMBER(10,2),
  platform_cut NUMBER(10,2), -- 30%
  worker_pay   NUMBER(10,2), -- 70%
  stripe_id    VARCHAR2(100),
  status       VARCHAR2(20),
  created_at   DATE DEFAULT SYSDATE
);

-- Reviews table
CREATE TABLE reviews (
  review_id  NUMBER PRIMARY KEY,
  job_id     NUMBER REFERENCES jobs(job_id),
  reviewer   NUMBER REFERENCES users(user_id),
  reviewee   NUMBER REFERENCES users(user_id),
  rating     NUMBER(1) CHECK (rating BETWEEN 1 AND 5),
  comment    CLOB,
  created_at DATE DEFAULT SYSDATE
);

Legal requirements
Protect yourself before anything else
🚨 URGENT — Get written agreement NOW
Before you write a single line of code get your deal confirmed in writing. Email reply from your friend minimum. Your dad can help draft something more formal. This is the most important thing on this entire list.
🚨 URGENT — Terms of Service and Privacy Policy before launch
Use termsfeed.com or privacypolicies.com to generate these for free. Your dad reviews them. Must be live on the platform before a single user signs up. Protects you if anything goes wrong.
Register the business in Alberta
Once you start making real money register through Alberta Business Registry (alberta.ca). Sole proprietorship (~$60) or Corporation (~$450). Your dad advises which is best. Don't do this until you have users and revenue.
Shareholders agreement (if Option 3)
If you go co-founder route — proper shareholders agreement before starting. Covers equity percentages, vesting schedule, decision making authority, what happens if someone leaves. SAIT legal clinic can help for free.
Vesting schedule
Shares earned over 2 years — protects everyone from someone leaving early and keeping equity for doing nothing. Standard: 6 month cliff then monthly vesting. Your dad knows this well.
GST registration
In Canada you must register for GST once you earn over $30,000/year. Under that it's optional. Keep this in mind as platform grows.
PIPEDA compliance — Canadian data privacy
Canada's privacy law requires protecting user data, clear privacy policy, and never selling user information. Firebase and Oracle have strong security built in. Privacy policy page must be live before launch.
Payment liability
Stripe handles payment security — you never store card numbers. Terms of Service must state platform is not responsible for disputes between posters and workers. Add a dispute resolution process.
Worker classification
Workers are independent contractors, not employees. Make this clear in Terms of Service. They are responsible for their own taxes on earnings. This is how TaskRabbit, Uber and Fiverr all operate.
Free legal help in Calgary
SAIT Legal Clinic (free for students) · University of Calgary free legal aid · Community Legal Clinics Alberta at legalaid.ab.ca · Use these before paying a lawyer.
