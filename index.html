Things that could trip you upBe careful about these
Stripe requires users to be 18+ — that's why 16-17 year olds need parent account with parent's payment method
Oracle reserved words — don't use MODE (use task_mode), COMMENT (use review_comment) as column names
Formspree limit is 50/month — create new form when approaching limit OR upgrade to paid plan
Reddit karma required — build karma by commenting before posting in r/Calgary and r/Edmonton
Don't promise things you can't deliver yet — say "coming soon" not "available now" in all promotions
Don't collect real payments yet — no Stripe integration until platform is actually built and tested
No adult content categories — strict ban, could get Stripe account terminated and platform shut down
Vetting workers is important — referral system + reviews + ID verification builds trust over time
People will try to pay in cash — payment must go through platform BEFORE task starts, no exceptions
People will try to go off platform — hide contact details, AI scan messages, warn then ban

Beta launch plan
How to go from coming soon to real platform
What is beta?Testing before full public launch
Beta = invite only testing phase — you control who gets access, fix bugs before everyone sees it, typically 50-200 users
Beta users are your waitlist people — they signed up early, they're invested, they'll give honest feedback
Beta is NOT public — no advertising, no press, just your trusted early users testing the platform
Phase 1 — Build MVP firstMinimum Viable Product — what you need before beta
Signup page working — Firebase Auth connected, users can create accounts ✅ page built, need Firebase
Login page working — users can log in and out securely ✅ page built, need Firebase
Post a task page — poster fills in task details, selects category, sets budget, posts task
Browse tasks page — workers see list of tasks near them, filter by category and distance
Apply for task — worker clicks apply, sends message and price offer to poster
Accept application — poster reviews applicants and accepts one worker
Payment through Stripe — poster pays into escrow, money held until task complete
Confirm task complete — poster confirms done, money releases to worker
Review system — both sides leave 1-5 star review after task
Basic dashboard — user sees their posted tasks, applications, earnings
Phase 2 — Beta launchInvite your waitlist — 50 people max
Email your waitlist — "QuickGigs beta is ready! You're one of 50 people getting early access. Here's your invite link."
Create beta invite codes — only people with codes can sign up, prevents random people joining before ready
Set up feedback form — Google Form asking beta users what works, what doesn't, what's missing
Create a beta WhatsApp group — stay in touch with beta users, answer questions, fix bugs fast
Watch for bugs daily — check Oracle database for errors, check Firebase for failed logins, fix fast
Run beta for 4-6 weeks — enough time to find all problems before public launch
Make sure at least 5-10 real tasks complete — real money, real workers, real posters, real reviews
